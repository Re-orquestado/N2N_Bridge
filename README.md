# N2N Bridge (Note-to-Note)

**Agentic AI Middleware for Adaptive Music Scores**

N2N Bridge is a Python middleware that translates game telemetry into structured musical intent, leveraging Large Language Models (LLMs) to generate deterministic MusicXML scores in real-time. It moves beyond traditional audio-looping systems, giving composers editable, high-quality musical scaffolding based on immediate game context.

Built for the **Kanz AI Hackathon**.

## Features

- **Game State Ingestion:** Reads live game data via JSON.
- **Cognitive Bridge:** Uses Groq API (Llama-3.3-70B) to map game parameters to musical parameters (tempo, key, instrumentation).
- **MusicXML Generation:** Outputs valid MusicXML files that can be directly opened and edited in any notation software (e.g., MuseScore, Sibelius).
- **Real-time Adaptation:** Designed for dynamic, reactive game soundtracks.

## Prerequisites

- Python 3.8+
- A Groq API Key (you can get one for free [here](https://console.groq.com/keys))

## Installation

1.  Clone this repository or download the files.
2.  Install the necessary dependencies:
    ```bash
    pip install groq python-dotenv
    ```

## Configuration

1.  Create a file named `.env` in the same directory as the script.
2.  Add your Groq API key to this file:
    ```text
    GROQ_API_KEY=your_actual_groq_api_key_here
    ```

## Usage

1.  Ensure your `telemetry.json` file contains the current game state you wish to translate.
2.  Run the main script:
    ```bash
    python hf_bridge.py
    ```
3.  The script will process the telemetry, call the Groq API, and generate a file named `output_scaffold.musicxml`.
4.  Open `output_scaffold.musicxml` in MuseScore 4 to view and edit the generated score.

## Project Structure

- `hf_bridge.py`: The main Python script handling telemetry reading, API calls, and XML generation.
- `telemetry.json`: Sample input file representing game state.
- `output_scaffold.musicxml`: The resulting musical score generated by the AI agent.
- `README.md`: This documentation file.
